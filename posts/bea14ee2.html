🧾 Chrome 扩展 manifest.json 配置详解

Chrome 浏览器扩展的配置文件是 manifest.json，它是每个扩展的核心配置文件，决定了扩展的基本信息、权限、功能等。

下面是对常见字段的详细解释：

🔧 基础配置

manifest_version

  • 类型 : 整数
  • 示例值 : 3
  • 作用 : 指定使用的 Manifest 版本。目前推荐使用版本 3（Manifest V3）。
  • 注意 : V3 和 V2 有较大差异，如后台脚本改为 Service Worker，不再支持某些注入方式。

name

  • 类型 : 字符串
  • 示例值 : "My First Extension"
  • 作用 : 插件名称，显示在浏览器插件管理页面和工具栏中。

version

  • 类型 : 字符串
  • 示例值 : "1.0" 或 "1.0.1"
  • 作用 : 插件版本号，用于更新检测和用户识别。
  • 建议 : 使用语义化版本号（Semantic Versioning）。

description

  • 类型 : 字符串
  • 示例值 : "一个简单的浏览器插件示例"
  • 作用 : 插件描述，显示在插件商店或管理页面中。

icons

  • 类型 : 对象

  • 示例值

  • 作用 : 插件图标，不同尺寸适配不同显示场景（如插件管理页、商店、弹窗等）。

  • 建议 : 提供多个尺寸（48x48, 128x128 等）以保证兼容性。

🧠 功能与权限配置

content_scripts

  • 类型 : 数组对象

  • 示例值 :

  • 作用 : 在匹配的网页中自动注入 JS 脚本和 CSS 样式。

  • 说明 :

    • matches: 匹配的 URL 模式。
    • js: 要注入的 JS 文件路径。
    • css: 可选，要注入的 CSS 文件路径。

background

  • 类型 : 对象

  • 示例值 :

    “background”: {

    “service_worker”: “background.js”

    }

  • 作用 : 定义后台服务脚本（相当于插件的“长期运行线程”）。

  • 注意 ：Manifest V3 中使用 service_worker 替代了之前的 background page。

host_permissions

  • 类型 : 数组字符串

  • 示例值 :

  • 作用 : 声明插件需要访问哪些网站的权限。

  • 适用场景 ：

    • 动态注入脚本（如 chrome.scripting.executeScript()）
    • 获取/修改 Cookie（如 chrome.cookies.get()）
    • 监听导航事件（如 chrome.webNavigation.onCompleted.addListener()）

permissions

  • 类型 : 数组字符串

  • 示例值

  • 作用 : 声明插件所需的其他权限。

  • 常用权限说明 ：

    • "cookies": 读写网页 Cookie
    • "storage": 使用 chrome.storage 存储数据
    • "activeTab": 操作当前激活的标签页
    • "webNavigation": 监听网页导航事件
    • "scripting": 允许动态注入脚本（V3 新增 API）

🖼️ 用户界面相关

action

  • 类型 : 对象

  • 示例值

  • 作用 : 定义点击插件图标的弹出窗口（Popup）和图标。

  • 说明 ：

    • default_popup: 弹出窗口的 HTML 页面。
    • default_icon: 插件图标的路径。

options_page

  • 类型 : 字符串
  • 示例值 : "options.html"
  • 作用 : 设置插件的设置页面（可选）。
  • 使用方法 ：在插件管理页面点击“详情” → “扩展选项”。

📦 其他配置项（高级）

  • 作用 : 允许外部网站与你的插件通信。

技巧 说明
测试扩展 在chrome://extensions/页面加载已解压的扩展包
查看后台日志 在扩展详情页点击“背景页”查看控制台输出
更新扩展 修改version后重新打包上传即可
